# 🎯 更简单的音频配置方案

## 两个核心问题的解决方案

---

## 问题1：豆包没有麦克风输入设置

### 解决方案：改系统默认麦克风

豆包会使用系统默认的麦克风，所以：

1. 打开 **系统设置 → 声音 → 输入**
2. 选择 **VB-Cable** 作为默认输入设备

✅ 这样豆包就会从VB-Cable听到TTS提示了！

---

## 问题2：自己听不到声音

### 方案A：使用OBS监听（推荐）

这是最简单的方法：

1. **系统设置**（只需要改这一次）：
   - **输出**：保持 **MacBook Air扬声器**（你自己能听到）
   - **输入**：选择 **VB-Cable**（豆包用这个）

2. **但是TTS声音怎么进VB-Cable呢？**
   
   我们需要让**终端的TTS输出到VB-Cable**，而不是扬声器。

3. **在终端中设置（每次运行脚本前执行）**：
   
   创建一个简单的启动脚本：

```bash
# 创建启动脚本
nano start_narrator.sh
```

填入以下内容：
```bash
#!/bin/bash
# 保存当前的音频输出
CURRENT_OUTPUT=$(SwitchAudioSource -c)

# 切换输出到VB-Cable
SwitchAudioSource -s "VB-Cable"

# 运行解说脚本
python3 doubao_narrator.py

# 脚本结束后切回原来的输出
SwitchAudioSource -s "$CURRENT_OUTPUT"
```

然后：
```bash
chmod +x start_narrator.sh
```

4. **同时，你自己怎么听到豆包和游戏声音？**
   
   用OBS的监听功能！

---

### 方案B：创建多输出设备（更完美，但稍复杂）

macOS可以创建一个同时输出到扬声器和VB-Cable的设备：

1. 打开 **音频MIDI设置**（在启动台搜索）
2. 点击左下角 **+** → **创建多输出设备**
3. 勾选：
   - ✅ MacBook Air扬声器
   - ✅ VB-Cable
4. 右键点击这个多输出设备 → **使用此设备用于声音输出**
5. 系统设置 → 声音 → 输出 → 选择这个多输出设备

✅ 这样TTS声音会同时到：
- 扬声器（你能听到一点点）
- VB-Cable（豆包能听到）

然后系统输入设为VB-Cable即可！

## 🚀 推荐的完整流程（方案A）

### 1. 安装SwitchAudioSource（方便切换音频设备）

```bash
brew install switchaudio-osx
```

### 2. 系统声音设置（一次性）

- **输出**：MacBook Air扬声器
- **输入**：VB-Cable

### 3. 创建启动脚本

创建 `run.sh`：
```bash
#!/bin/bash
echo "🎤 准备启动AI解说..."
echo "⏳ 切换音频输出到VB-Cable..."

# 切换输出到VB-Cable
SwitchAudioSource -s "VB-Cable"

echo "✅ 音频已切换"
echo "🎮 5秒后启动..."
sleep 5

# 运行解说脚本
python3 doubao_narrator.py

# 脚本结束后切回扬声器
echo "🔄 恢复音频输出..."
SwitchAudioSource -s "MacBook Air扬声器"
echo "✅ 完成！"
```

### 4. OBS设置

- 采集豆包客户端音频
- 采集游戏音频
- 都开启"监听并输出"

### 5. 每次使用时

```bash
# 打开OBS，启动监听
# 然后运行：
chmod +x run.sh
./run.sh
```

## 🎵 音频流向（方案A）

```
系统输入 → VB-Cable → 豆包听到
     ↑
     │
终端TTS ──→ VB-Cable（运行时临时切换）

系统输出 → 扬声器 → 你听到
     ↑
     │
OBS监听 ──→ 你听到豆包和游戏
```
