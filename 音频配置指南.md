# 🎵 音频路由配置指南

## 目标
- **TTS提示语音** → 只让豆包听到，**不让观众听到**
- **豆包的解说语音** → 让观众听到
- **游戏/系统音效** → 让观众听到

---

## 📋 音频设备说明

你现在有这些音频设备：
1. **MacBook Air麦克风** - 你的物理麦克风
2. **OrayVirtualAudioDevice** - 虚拟音频设备
3. **VB-Cable** - 虚拟音频线缆（我们主要用这个）

---

## 🔧 详细配置步骤

### 第一步：配置系统音频输出（让TTS走VB-Cable）

1. 打开 **系统设置** → **声音**
2. 点击 **输出** 标签
3. 选择 **VB-Cable** 作为输出设备
   - 这样终端播放的TTS声音就会进入VB-Cable
   - ⚠️ 注意：此时你的扬声器不会发出声音了

### 第二步：配置豆包的麦克风输入

1. 打开豆包桌面客户端
2. 发起语音通话
3. 在通话设置中，将 **麦克风输入** 选择为 **VB-Cable**
   - 这样豆包就能听到VB-Cable里的TTS提示语音了

### 第三步：配置OBS音频采集

打开OBS Studio，按以下配置：

#### 1. 添加豆包解说音频（观众能听到）
- 点击 **来源** → **+** → **Application Audio Capture (BETA)**
- 命名为 "豆包解说"
- 选择 **豆包客户端** 作为应用
- 确定

#### 2. 添加游戏/系统音频（观众能听到）
- 点击 **来源** → **+** → **Application Audio Capture (BETA)**
- 命名为 "游戏音效"
- 选择你的游戏应用
- 确定

#### 3. （可选）添加你的麦克风（如果需要）
- 点击 **来源** → **+** → **音频输入采集**
- 选择 **MacBook Air麦克风**
- 可以在混音器中静音/取消静音

### 第四步：监听设置（让你自己能听到声音）

由于系统输出设为VB-Cable，你听不到声音了，需要设置OBS监听：

1. 在OBS的 **混音器** 面板
2. 找到 "豆包解说" 和 "游戏音效"
3. 点击每个音频源旁边的 ⚙️ 图标
4. 选择 **高级音频属性**
5. 在 **音频监听** 列，选择：
   - "豆包解说" → **监听并输出**
   - "游戏音效" → **监听并输出**
6. 将OBS的 **监控设备** 设置为你的扬声器/耳机

1. **运行测试脚本**：
   ```bash
   python3 test_tts.py
   ```
   - 你应该听不到声音（因为走了VB-Cable）
   - 豆包应该能听到并回复

2. **打开OBS**：
   - 看混音器是否有信号
   - 点击 **开始推流/录制** 测试

3. **启动解说脚本**：
   ```bash
   python3 doubao_narrator.py
   ```

## ❓ 常见问题

**Q: 我自己听不到声音了怎么办？**
A: 在OBS的高级音频属性中开启"监听"，OBS会把声音输出给你。

**Q: 观众听到了TTS提示音？**
A: 检查OBS有没有采集到VB-Cable，不要把VB-Cable加入OBS来源。

**Q: 豆包听不到TTS提示？**
A: 确认豆包的麦克风输入选的是VB-Cable。
